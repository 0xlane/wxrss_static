Code review 代码审查不仅仅是为了更正代码错误，更是为了共享知识、建立统一的代码指导标准。 如果可以，应该月度组织一场“代码审查会议”。 （当然，我才不是为了开会的时候来点下午茶。） 4. 克制你的注释 注释的目的是补充代码所不能表达的意图。 如果能通过调整代码结构、重命名函数和变量就能解释清楚的就不要写注释。 注释就像给代码打补丁，不需要“补满”，只需要“补漏”。 5. 早部署、常部署 开发中，完成一些功能我们就部署联调一次，确保我们工作是有效且可控的。 做到早部署、常部署，问题才能早发现，早解决。 别总想着最后拿出来一鸣惊人， 哪怕是科比或詹姆斯都不能保证投进每个“压哨球”。 6. DRY（Don't Repeat Yourself） “不要重复自己”，是软件开发中一个重要的原则，旨在减少重复性的代码，提高代码的可维护性和可重用性。 让代码更DRY一点，成为真正的“干货”。 7. 找备胎 软件设计中，构建模块时的第一个想法不太会是完美的设计。 多做些备选方案，选择不同方法进行对比，往往会启发更多。 写程序不是谈恋爱，多找几个“备胎”，才能撞出“最佳CP” 8.数据“控” 数据是系统的核心，数据比代码更精炼，也更好理解。 要想读懂程序，最难的就是厘清哪些数据之间是相互关联的。 相较优美的代码，数据的清晰且合理更为关键，优秀的程序员，一定是个认真的数据“控” 距离100条又近了一步，还有什么是你觉得程序员一定要知道的？评论区告诉我哦~
