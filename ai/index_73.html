  • Claude Code 怎么使用最省？ 刚开始我选择了Pro计划，觉得每月20美元很划算，结果发现每5小时只能发送10-40个提示，而且还不能用Opus 4模型。 用了一周就发现完全不够，经常卡在使用限制上，开发效率反而降低了。后来咬牙升级到Max计划，虽然每月要花100美元，但算下来一个24/7工作的智能编程助手成本远比请人工便宜，这笔投资确实值得。 现在的模型使用策略很明确：日常开发全部用Sonnet 4，这个模型在性能和成本之间达到了最佳平衡，只有遇到复杂的架构设计或需要深度代码分析时才会切换到Opus 4。 对于简单的代码格式化、变量重命名这类重复性工作，我都用Windsurf免费的模型，比方说<a class="normal_text_link" data-itemshowtype="" target="_blank" href="https://mp.weixin.qq.com/s?__biz=MzIzMzQyMzUzNw==&mid=2247505524&idx=1&sn=5806e5e697bfb8b70436f17df4ec2cb7&scene=21#wechat_redirect">Grok Code</a>，或者成本更低的GPT-5。 还有我发现集中化地批量处理相同类型的任务也会比零散处理更省钱，比如在一个任务里面处理lint的报错，或者单独完成拆分重构组件的任务。 最省钱技巧是频繁使用`/clear`命令。我现在养成了习惯，每次开始新任务前就清理对话历史，这样避免了大量无关信息消耗token。刚开始我不知道这个，让无关的历史对话一直占用着配额，钱花得冤枉。 另外`/compact`命令也很有用，可以战略性地压缩上下文，只保留当前任务相关的信息。 再就是要精确选择需要分析的文件。虽然Claude Code支持百万token的上下文窗口，但一次性加载整个大型代码库会让成本飙升。现在我都用@标签有针对性地引用特定代码段，既提高了效率又控制了成本。 监控使用情况也很重要的，我会经常用`/status`和`/cost`命令检查，就像开车时看油表一样。装了ccusage这个第三方工具，能看到详细的使用分析和成本分解，帮我找出哪些操作最耗token。 像类似于Git提交这种非编码业务类的操作，我是绝对不会在CC里面完成的，都是使用免费Gemini CLI制作成工作流，量大管饱分析准确：<a class="normal_text_link" data-itemshowtype="" target="_blank" href="https://mp.weixin.qq.com/s?__biz=MzIzMzQyMzUzNw==&mid=2247502174&idx=1&sn=32f0494149360829be896b5e01079ea6&scene=21#wechat_redirect">Gemini CLI高级使用技巧和案例演示及常见问题解答</a> 最后就是配搭使用，不要怼着Claude Code一头薅，而是搭配着Windsurf来使用：<a class="normal_text_link" data-itemshowtype="" target="_blank" href="https://mp.weixin.qq.com/s?__biz=MzIzMzQyMzUzNw==&mid=2247505008&idx=1&sn=e4ee8a96c8cf52cab5c1e1771b519014&scene=21#wechat_redirect">Windsurf的重生！</a>，比方说代码重构，让CC来制作任务TodoList ，然后再跑到Windsurf里面来具体执行，基本上消耗一个点就可以解决。 更多的CC使用技巧欢迎下方留言讨论。
